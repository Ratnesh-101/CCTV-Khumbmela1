CCTV Risk Intelligence Stack

Real-world anchor: Maha Kumbh Mela–scale gatherings — millions on ghats & corridors, fixed CCTV on bridges and routes.
This stack fuses perception signals so one operator can prioritize feeds before a situation escalates (crowd surge, dispute, unattended object, traffic incident).
Tech: Python · YOLOv8 · OpenCV · PyTorch · Streamlit · LangChain · FAISS · scikit-learn · Gymnasium

Why Kumbh Mela is the right example

  • Scale: Extremely high person density; small delays in noticing crowding or commotion matter.
  • Infrastructure: Mela authorities deploy hundreds of fixed cameras — not body-worn — matching our “fixed CCTV” design.
  • Operational reality: Control rooms see too many tiles; raw YOLO boxes don’t say “getting worse over 30s” or “which cue to trust.”
  • Use we support: Scene-level risk ranking + playbook text (RAG) for SOP recall — not mass facial recognition of pilgrims.

What problem this pipeline solves

  • Fragmentation: Detections, motion, and heuristics live in separate heads — operators get noise, not a single trend line.
  • No fused score: We produce a transparent 0–100 risk scalar plus named flags (disturbance_motion, dense_crowd, …) per frame.
  • No escalation hint: A small tabular Q-policy suggests a discrete action class (demo) to structure alert debouncing.
  • No procedure memory: RAG retrieves crowd / lost-article / evacuation snippets for the operator console.
  • Kumbh mapping: Crowd + motion + proximity ≈ pressure on corridors; unattended bags ≈ security SOP; optional audio spike ≈ incident cue (demo-level).

Core loop (simplified code)

This is the actual orchestration pattern inside app.py / process_video.py:

while frames:
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    sig = analyzer.analyze_frame(frame, prev_gray, gray, tracker)
    rs = compute_risk(
        person_count=sig.person_count,
        motion_score=sig.motion_score,
        proximity_cluster_score=sig.proximity_cluster_score,
        abandoned_seconds=sig.abandoned_max_seconds,
        weapon_like_score=sig.weapon_like_score,
        fight_like_score=sig.fight_like_score,
        theft_like_score=sig.theft_like_score,
        accident_like_score=sig.accident_like_score,
        missile_like_score=sig.missile_like_score,
        drone_like_score=sig.drone_like_score,
        wifi_rf_anomaly_score=wifi_score,
        ...
    )
    act_id, act_name = rl_action(rs.total, cooldown_state, Q_table)
    writer.write(draw_overlay(frame, sig, rs.total, rs.labels))

Perception & fusion (technical)

  • YOLOv8 (VideoAnalyzer): persons, bags, optional custom classes (fight, accident, weapon, …) via name sets.
  • Optical flow (motion_from_flow): Farneback on masked person ROIs → motion_score.
  • Proximity (proximity_score): pairwise person centroids → crowding cue (Kumbh: corridor crush proxy).
  • Bags (SimpleBagTracker): idle time + owner distance → abandoned_max_seconds.
  • Fusion (compute_risk): weighted sum of normalized components → total + thresholded labels.

RL · RAG · optional sensors

  • RL: risk_bucket(rs.total) discretizes score; Q-table outputs escalation hint with cooldown debouncing.
  • RAG: LangChain + FAISS + embeddings — operator queries (“unattended bag procedure”) without leaving the console.
  • Wi-Fi tab: Isolation Forest on flow features — separate from video; demo for anomaly “cue” fusion.
  • Audio tab: Transient energy heuristic → acoustic_threat channel (not a gunshot classifier).

Outputs for a Mela control room

  • Annotated MP4/AVI: boxes + [tags] + green risk-flag strip — quick visual audit.
  • Time-series: risk vs time + stacked components — “which camera is heating up.”
  • CSV / dataframe tail: per-frame log for post-incident review or threshold tuning.
  • Geotagged alerts (simulated): fixed camera lat/lng in UI — maps to physical sector on the ground.

Honest limits @ Kumbh-scale deployment

  • Not a certified crowd-safety system — heuristics false-alarm on normal density; human confirmation required.
  • Train on Mela/CCTV data for fight, accident, stampede-like motion if those labels exist; COCO alone is insufficient.
  • Privacy & law: use for public-safety operations with governance; avoid repurposing for individual tracking at scale.
  • Engineering next: edge inference, NTP-synced multi-camera correlation, calibrated thresholds per sector, formal incident workflow.
